# PRD — pathguard

## 무엇을

클라우드/NAS로 동기화하는 로컬 폴더의 **파일명·경로 길이**를 감시해,
한글 조합형(NFD) 확장 때문에 **동기화가 깨질 위험이 있는 파일을 미리 잡아내는** 읽기전용 도구.
동기화 앱에 종속되지 않는다 (Synology Drive · Dropbox · Nextcloud · OneDrive · iCloud 등).

Go로 구현: **맥/윈도우 트레이·메뉴바 상주 앱**(`cmd/pathguard-gui`)과 **헤드리스 CLI**(`cmd/pathguard`).
초기 프로토타입이던 `pathguard.py`는 Go CLI로 대체·제거됨(T-0016).

## 왜

- POSIX/ btrfs 한계 `NAME_MAX 255` / `PATH_MAX 4096` 은 **문자 수가 아니라 UTF-8 바이트 수**다.
- 한글은 정규화 형태에 따라 바이트가 달라진다: NFC(완성형) 자당 3B, **NFD(조합형) 자당 6~9B**.
- macOS는 새 한글 파일명을 NFD로 저장하고, 클라이언트/OS마다 정규화가 달라진다.
- 즉 **지금 맥에선 짧아 보여도** 어디선가 NFD로 풀리면 한계를 넘겨 동기화가 조용히 실패한다.
- 그래서 **항상 NFD(최악치) 바이트**로 환산해 사전 경고가 필요하다.
- 정규화(NFC 변환)로는 못 고친다. **유일한 해법은 이름을 짧게 rename** 하는 것 → 사람이 알아채도록 알림이 핵심.

## 사용자 & 시나리오

- 1차 사용자: 클라우드/NAS로 문서·노트 폴더를 동기화하는 맥·윈도우 사용자.
- 시나리오: 앱이 트레이/메뉴바에 상주 → 주기적으로 감시 폴더 스캔 → 초과·경고 발견 시 **트레이 아이콘이 warn 상태로 변경** → 사용자가 메뉴로 상세 확인 → 문제 파일 rename.

## 성공 기준

1. **정확성**: NFD 최악치 기준으로 NAME_MAX/PATH_MAX 초과·경고를 빠짐없이 잡는다(오탐 최소).
2. **설치 편의**: 맥(인텔/애플실리콘)·윈도우 모두 쉽게 설치. 윈도우는 **의존성 없는 단일 exe**.
3. **상주 UX**: 트레이/메뉴바 아이콘 상주, 클릭 시 메뉴로 감시 폴더·주기·시각 설정 가능.
4. **알림**: 위험 발견 시 아이콘 상태 변화로 즉시 인지. (옵션) OS 네이티브 배너 병행.
5. **안전**: 파일을 절대 수정/이동/삭제하지 않는 **읽기전용**.
6. **설정**: 사람이 손으로 읽고 고칠 수 있는 **YAML** 설정 파일.

## 범위 밖 (Non-goals)

- 파일 자동 rename/정규화 변환 (읽기전용 원칙 위반).
- 원격 서버 측 에이전트 설치 (로컬에서 동기화 폴더만 본다).
- Linux 데스크톱 상주(당장은 아님, CLI는 동작).
- 실시간 파일시스템 감시(inotify/FSEvents)는 초기 범위 밖 — **폴링 주기** 방식으로 시작.
